El tipo de NaN

Resultado

NaN es un dato de tipo number debido a que representa los números que no tienen una representación en un contexto numérico

Decimales de precisión

Resultado

JavaScript tiene 16 decimales de precisión, si supera dicho límite, redondea.

Suma de arrays vacios

Resultado

El carácter "+" actua como operador de concatenación en este contexto, por lo que implícitamente los arrays se convierten en cadenas. Al estar vacios, devuelven la cadena vacia, y la suma de dos cadenas vacias es una cadena vacia.

Objeto a cadena

Resultado

La representacion de un objeto como cadena es un literal [object Object]. Si a un array vacio le sumas un objeto, el array se convertirá implicitamente en cadena vacia y se concatenará con el literal quedando igual.

true en contexto numerico

Resultado

true en un contexto numérico se convierte en 1. Por lo que simplemente se están sumado y restando "unos".

Comparación true = 1

Resultado

Si comparamos true == 1, nos devuelve true. Sin embargo al comparar true === 1, nos devuelve false, ya que se comparan los tipos antes de realizar cualquier conversión.

Comparación [] = 0

Resultado

En contexto numerico, un array vacio devuelve 0. Sin embargo al comparar [] === 0, nos devuelve false, ya que se comparan los tipos antes de realizar cualquier conversión.

Comparación numero +- cadena

Resultado

El caracter "+" actua como operador de concatenación si al menos uno de los dos operandos es una cadena. El caracter "-" solo tiene contexto numérico, por lo que aquí, la cadena "1" se convierte a número.
